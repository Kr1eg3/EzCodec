# Fermat Spiral Color Encoding

## Идея

Единый вещественный параметр `t ∈ [0, 1]` кодирует цвет через спираль Ферма,
вписанную в HSL-диск. Один float → любой цвет.

---

## Цветовая модель

HSL-диск: плоский срез HSL-пространства при фиксированной яркости `L`:

- **угол** φ → Hue (цветовой тон, 0…360°)
- **радиус** r → Saturation (насыщенность, 0…1)
- **L** = константа → Lightness (яркость)

---

## Прямая функция: t → RGB

### Шаг 1. Спираль Ферма → HSL

```
S = sqrt(t)          // радиус → Saturation
H = frac(N * t)      // угол   → Hue ∈ [0, 1)
L = const            // яркость, рекомендуется 0.5
```

Где:
- `N` — число витков спирали (параметр разрешения палитры)
- `frac(x) = x mod 1`

### Почему sqrt(t)?

Площадь кольца радиуса r и ширины dr:

```
dA = 2π r dr
```

Чтобы каждый шаг dt покрывал одинаковую площадь диска:

```
dA/dt = const
⟹ r · dr/dt = const
⟹ d(r²)/dt = const
⟹ r² = t
⟹ r = sqrt(t)
```

Это **математически необходимое** условие равномерного покрытия HSL-диска.

### Шаг 2. HSL → RGB

Стандартная замкнутая формула (без таблиц):

```
a = S * min(L, 1 - L)

для k ∈ {0, 8, 4}:
    λ = fmod(k + 12 * H, 12)
    channel = L - a * clamp(min(λ - 3, 9 - λ), -1, 1)

R = channel(k=0)
G = channel(k=8)
B = channel(k=4)
```

---

## Параметры

| Параметр | Диапазон | Роль |
|----------|----------|------|
| `t`      | [0, 1]   | единственный вход — кодирует цвет |
| `N`      | 1…∞      | число витков = разрешение палитры |
| `L`      | [0, 1]   | яркость (рекомендуется 0.5) |

**Чем больше N:**
- Меньше расстояние между соседними витками (~1/N)
- Точнее адресация любого цвета на диске
- Аналог битовой глубины

---

## Реализация на C

### Прямая функция: `fermat_color(t, N, L) → RGB`

```c
#include <math.h>

typedef struct { unsigned char r, g, b; } RGB;

/* t ∈ [0,1] — кодовое число цвета
   N         — число витков (разрешение палитры)
   L ∈ [0,1] — яркость (0.5 = насыщенный) */
RGB fermat_color(float t, int N, float L)
{
    /* 1. Спираль Ферма → HSL */
    float S = sqrtf(t);
    float H = fmodf((float)N * t, 1.0f);

    /* 2. HSL → RGB */
    float a = S * fminf(L, 1.0f - L);

    int   k[3] = {0, 8, 4};
    float ch[3];
    for (int i = 0; i < 3; i++) {
        float lam = fmodf((float)k[i] + 12.0f * H, 12.0f);
        float val = lam - 3.0f;
        if (9.0f - lam < val) val = 9.0f - lam;
        if (1.0f       < val) val = 1.0f;
        if (-1.0f      > val) val = -1.0f;
        ch[i] = L - a * val;
    }

    RGB out;
    out.r = (unsigned char)(ch[0] * 255.0f + 0.5f);
    out.g = (unsigned char)(ch[1] * 255.0f + 0.5f);
    out.b = (unsigned char)(ch[2] * 255.0f + 0.5f);
    return out;
}
```

### Обратная функция: `fermat_color_inverse(RGB, N) → t`

Принцип: `t` восстанавливается из двух независимых источников:

- **из Saturation**: `t_S = S²` — точно, без двусмысленности
- **из Hue**: `t_H = (H + k) / N`, где `k` — номер витка (0…N-1)

Алгоритм: подобрать `k` так, чтобы `t_H` был ближайшим к `t_S`,
затем уточнить взвешенным средним.

```c
static void rgb_to_hsl(unsigned char r, unsigned char g, unsigned char b,
                       float *H, float *S, float *L)
{
    float R = r / 255.0f, G = g / 255.0f, B = b / 255.0f;
    float cmax = R, cmin = R;
    if (G > cmax) cmax = G;  if (B > cmax) cmax = B;
    if (G < cmin) cmin = G;  if (B < cmin) cmin = B;
    float delta = cmax - cmin;

    *L = (cmax + cmin) * 0.5f;
    *S = (delta < 1e-6f) ? 0.0f
                         : delta / (1.0f - fabsf(2.0f * *L - 1.0f));

    if (delta < 1e-6f) { *H = 0.0f; return; }

    float h;
    if      (cmax == R) h =        (G - B) / delta;
    else if (cmax == G) h = 2.0f + (B - R) / delta;
    else                h = 4.0f + (R - G) / delta;
    h /= 6.0f;
    *H = h - floorf(h);
}

/* Возвращает t ∈ [0,1].
   N должен совпадать с использованным при кодировании. */
float fermat_color_inverse(unsigned char r, unsigned char g, unsigned char b,
                           int N)
{
    float H, S, L;
    rgb_to_hsl(r, g, b, &H, &S, &L);

    float t_S = S * S;

    float best_t = t_S, best_d = 1e9f;
    for (int k = 0; k < N; k++) {
        float t_H = (H + (float)k) / (float)N;
        float d   = fabsf(t_H - t_S);
        if (d < best_d) { best_d = d; best_t = t_H; }
    }

    /* Взвешенное среднее: у краёв доверяем Hue, у центра — Saturation */
    float t = (1.0f - S) * t_S + S * best_t;
    if (t < 0.0f) t = 0.0f;
    if (t > 1.0f) t = 1.0f;
    return t;
}
```

## Точность round-trip

| Зона диска     | Источник ошибки              | Точность          |
|----------------|------------------------------|-------------------|
| Край (S → 1)   | квантование 8-bit            | ~1/(255·N)        |
| Центр (S → 0)  | Hue неопределён              | деградирует       |
| t = 0 (центр)  | особая точка спирали         | не восстановим    |

**Сингулярность**: при `t = 0` все точки сходятся в центре диска (серый цвет).
Hue теряется необратимо. Обойти без дополнительного канала данных невозможно.

---

## Свойства

- **Замкнутая формула** — нет таблиц, нет интерполяции
- **Равномерное покрытие по площади** — математически доказано через условие dA/dt = const
- **N как разрешение** — аналог битовой глубины: чем больше N, тем точнее адресация
- **Зависимости** — только `<math.h>`, никаких внешних библиотек
- **Производительность** — цикл из 3 итераций, разворачивается компилятором

---

## Пример использования

```c
// Кодирование
RGB c = fermat_color(0.42f, 12, 0.55f);
printf("rgb(%d, %d, %d)\n", c.r, c.g, c.b);
// → rgb(?, ?, ?)

// Декодирование (round-trip)
float t_recovered = fermat_color_inverse(c.r, c.g, c.b, 12);
printf("t = %.4f\n", t_recovered);
// → t ≈ 0.4200
```
